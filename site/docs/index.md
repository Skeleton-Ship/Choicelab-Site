If you've ever written (or just been interested in) interactive stories, there's a chance you've been introduced to tools like [Twine](https://twinery.org). Maybe there are other indie game development tools you've learned about, like [Bitsy](https://www.bitsy.org) or [Pixelorama](https://orama-interactive.itch.io/pixelorama).

These tools are popular because they're approachable, fun, and _specific_. There are big, monolithic, many-featured game engines that do what they do—but those tools have a steep learning curve, and they can be expensive.

A good tool, the kind that makes you excited to create things, is usually a specific one. Choicelab's primary goal is to be **a very nice tool for developing multimedia-rich interactive stories**. Instead of textual interactive fiction, Choicelab uses _audio_ and _video_ as the building blocks for its narratives. “Multimedia-rich” is the key adjective.

There are lots of textual interactive fiction tools that let you make great narrative experiences. But if you want to tell stories using motion and sound design—that perhaps bridge the gap between films, serialized audio, and games—we're building Choicelab for you.
